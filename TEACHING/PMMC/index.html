Course objectives and examination rules

Some time ago a friend of mine told me the following ".... parallel programming is for a few, kernel programming is for a few of them ....". In the end I fully agreed upon this concept, definitely supporting the idea that the deepest approach to the explanation of parallelism and concurrency is one based on both user and kernel space operations. This is the reason why this course on multi/many-core programming, beyond presenting concepts on the relation between hardware parallelims vs software design, is also focused on kernel level software.

Parallelism in the hardware---either implicit or explicit---will be analyzed by the side of its impact on advanced design/implementation methods and techniques for modern system/kernel level software. The topics dealt with by the course are bound to case studies oriented to x86 compliant processors and LINUX systems. In any case they can be easily transposed to other hardware/software systems. The course requires basic knowledge on computer architecture, on the structure and functionalities of operating systems, and knowledge on C/ASM programming.

The lecture flow will be based on presenting concepts in combination with running examples. Both the slides and the software examples will be made available while delivering the course, clearly in advance to discussing them at any lecture. The slides as well as the software examples can be accessed and downloaded from links posted in the last section of this Web page

The examination consists of a set of questions and additionally requires the development of software sub-systems (a project) to be embedded within the LINUX kernel, whose specification for the academic year 2017/2018 will be posted during the course delivery. Each student is supposed to finalize both these parts of the examination within the same accademic year. The delivery of sofwtare project does not need to coincide with the scheduled examination dates. Rather, students can deliver their projects at any student reception (see this link).

List of planned topics (2017-2018)

  • Hardware review and synchronization/coordination concepts
    • Pipelining and superscalar processors
    • Speculative hardware
    • Multi-processors and multi-cores
    • Physical memory organization
    • Memory coherency and consistency
    • Hardware synchronizaiton support
    • Linearizability and thread coordination
    • Vectorization

  • System level software - with case studies on Linux kernel 2.4/2.6/3.x/4.x
    • Operating system-kernel surface and kernel programming basics
    • Loadable modules
    • Scalable memory management
    • Advanced (HW/SW) interrupt architectures
    • System interactions with the caching hierarchy
    • Scalable task/thread management
    • VFS and drivers
    • Operating System security aspects
